# 🚀 快速启动步骤

## ⚡ 方案 A：使用简化版 SQL（推荐新手）

这个方案移除了外键约束，更容易成功执行。

### 第 1 步：初始化数据库（2 分钟）

```bash
# 1. 打开命令行，登录 MySQL
mysql -u root -p

# 2. 删除旧数据库（如果有）
DROP DATABASE IF EXISTS lab_items_management;

# 3. 执行简化版 SQL 文件
source c:/Users/north/Desktop/1/database_simple.sql;

# 4. 验证
USE lab_items_management;
SHOW TABLES;
```

**预期结果：** 看到 3 张表（user, category, system_config）

### 第 2 步：启动后端（3 分钟）

```bash
# 1. 打开新的命令行窗口
cd c:\Users\north\Desktop\1\lab-items-backend

# 2. 修改配置文件（重要！）
# 用记事本打开 src/main/resources/application.yml
# 找到数据库密码，修改为你的 MySQL 密码

# 3. 启动后端
mvn spring-boot:run
```

**等待看到：**
```
实验室物品管理平台启动成功！
API 文档：http://localhost:8080/api/swagger-ui.html
```

### 第 3 步：启动前端（2 分钟）

```bash
# 打开另一个新的命令行窗口
cd c:\Users\north\Desktop\1\lab-items-frontend

# 首次运行需要安装依赖
npm install

# 启动前端
npm run dev
```

**等待看到：**
```
➜  Local:   http://localhost:3000/
```

## 🐛 常见问题解决

### 问题 1：数据库连接失败

**错误：** `Could not open JDBC Connection`

**解决：**
```bash
# 1. 检查 MySQL 服务
net start mysql

# 2. 修改密码
# 编辑 lab-items-backend/src/main/resources/application.yml
spring:
  datasource:
    password: your_password  # 改为你的密码
```

### 问题 2：端口被占用

**错误：** `Port 8080 was already in use`

**解决：**
```bash
# 查找并结束进程
netstat -ano | findstr :8080
taskkill /F /PID <进程 ID>
```

### 问题 3：前端 npm install 失败

**错误：** 网络超时

**解决：**
```bash
# 使用淘宝镜像
npm config set registry https://registry.npmmirror.com
npm install
```

### 问题 4：SQL 执行错误

如果使用 `database_init.sql` 有错误，请使用简化版：
```bash
mysql -u root -p < database_simple.sql
```
